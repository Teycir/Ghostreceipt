# Repository Metadata Checklist

Use this checklist for GitHub-side metadata that is not fully captured in code.

## Discoverability

- [ ] Fix GitHub topic typo: replace `payement` with `payment`.
- [ ] Ensure core topics are present and ordered:
  - [ ] `zero-knowledge`
  - [ ] `zkp`
  - [ ] `privacy`
  - [ ] `ethereum`
  - [ ] `bitcoin`
  - [ ] `payment`
  - [ ] `nextjs`

## Live Demo Publication

- [ ] Publish the GhostReceipt live demo URL.
- [ ] Add the same URL to:
  - [ ] GitHub repo "About" website field.
  - [ ] `README.md` main header section.
  - [ ] Release notes for the first public tag.

## Release Metadata

- [ ] Create first tagged release (`v0.1.0` or chosen version).
- [ ] Attach release notes summary and known limitations.
- [ ] Link security/trust docs in release notes:
  - [ ] Oracle trust model
  - [ ] Security runbook
  - [ ] Trusted setup provenance record

## Ownership

- [ ] Assign maintainer responsible for repo metadata hygiene.
- [ ] Set recurring quarterly check for topic/demo/release metadata consistency.
